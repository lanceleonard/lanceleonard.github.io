Editorial resources for writers, editors, and wearers of multiple hats.

I recently asked an online content group for links to "style guides." There were several responses, including many links to "design guides," as in guidelines for interface design: layout, colors, fonts, and so on. Interesting stuff to be sure, but not precisely what I was looking for.

I was looking for editorial style guides, such as The Chicago Manual of Style (CMoS), AP Stylebook, and so on. Not that these aren't excellent resources, but if you're on a budget (as I was), you might be interested in freely available resources.

If you're interested in editorial style guides, here's what I've gathered so far:

Editorial style resources

- 18F [Content Guide](https://pages.18f.gov/content-guide/): A "clear writing" resource.

- American Physiological Society (APS) [Manual of Style](http://www.apsstylemanual.org/).  An [earlier version](http://www.apsstylemanual.org/oldmanual/index.htm) is also available and worth reviewing.

- [AP vs Chicago](http://apvschicago.com) compares differences between the _AP Stylebook_ and the _Chicago Manual of Style_.

- [Apple Style Guide](https://help.apple.com/applestyleguide).

- [Blue Book of Grammar](http://www.grammarbook.com/):  Extensive, but a bit hard to read.

- Chicago [Manual of Style Online](http://www.chicagomanualofstyle.org): Subscription required, but search occasionally returns useful tidbits.

- [Daily Writing Tips](http://www.dailywritingtips.com/size-matters/) offers advice for common (and uncommon) writing questions.

- Dojo (JavaScript library) [documentation style](http://dojotoolkit.org/reference-guide/1.10/developer/styleguide.html#developer-styleguide) guide.

- [English Phrases](http://www.phrases.org.uk/) explains the meaning and origins of common phrases and iodoms. (UK).   

- Google's [Developer Documentation](https://developers.google.com/style) Style Guide.  For user experience (UX) guidelines, see their [Material Style Guide](https://material.io/guidelines/style/writing.html).

- [GrammarCheck.net](http://www.grammarcheck.net) provides helpful infographics and grammar review. 

- Microsoft [Writing Style Guide](https://docs.microsoft.com/style-guide/welcome) represents the latest ideas and practices used in docs.microsoft.com; it supersedes and replaces the _Microsoft Manual of Style_ (MMoS) and the _Microsoft Guide to Technical Publications_ (MSTP).  If you're familiar with the earlier works, take time to review the new guidelines. 

- Microsoft [REST API Guidelines](https://github.com/Microsoft/api-guidelines/blob/master/Guidelines.md). 

- National Archives (NARA) - Principles for [Plain Language](http://www.archives.gov/open/plain-writing/10-principles.html) and [Style Guide (PDF)](https://www.archives.gov/files/open/plain-writing/style-guide.pdf).

- Perdue University [Online Writing Library (OWL)](https://owl.english.purdue.edu/owl/)

- [The Punctuation Guide](http://www.thepunctuationguide.com/index.html): An interesting interface, but a bit light on content.

- [Rackspace Documentation Guide](http://rackerlabs.github.io/docs-rackspace/style-guide/basic-writing-guidelines.html) offers sage advice while breaking a other few rules.  

- Salesforce Documentation [Style Guide](https://developer.salesforce.com/docs/atlas.en-us.salesforce_pubs_style_guide.meta/salesforce_pubs_style_guide/overview.htm).  

- Washington State University's [Common English Errors](http://public.wsu.edu/~brians/errors/errors.html#errors). 

- [Writing Explained](http://writingexplained.org/) is a very interesting resource that combines information from multiple sources, including summarizing rules from _the Chicago Manual of Style_, _AP Stylebook_, and other sources.  Highly recommended.

Editorials and guidance from other technical writers:

- James Cooke: [The Communication Illusion](http://jamescooke.info/api-documentation-and-the-communication-illusion.html).

- James Cooke: REST API [documentation templates](https://github.com/jamescooke/restapidocs) in Markdown. 

- Tom Johnson introduces [Simplified Technical English](http://idratherbewriting.com/2017/01/25/hyperste-simplified-technical-english-asd-ste100/).

- Sean Madsen's [Elegant Documentation Principles](https://github.com/seanmadsen/elegant-docs).

- Bob Reselman's [7 rules for creating world class docs](http://www.developer.com/tech/article.php/3848981/The-7-Rules-for-Writing-World-Class-Technical-Documentation.htm).

If you have other resources, please feel free to share them. I'll update these lists from time to time.
